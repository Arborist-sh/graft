## Landmines / don't-redo (settled — reference, don't relitigate)
- **GUI shells the brew `graft` (`/opt/homebrew/bin`).** It's still **0.4.0** — so the orphan-
  exclusion, nest provisioning-status, `-t` PTY shell, and inspect features **won't work in the
  GUI until `scripts/dev-link.sh`** swaps in the dev CLI. (Start/Stop/Window/Remove use `tart`
  directly and work regardless.)
- **GitHub can't** retrieve an existing App's private key, list owned Apps, or mint a key via API
  — manifest flow (browser) is the only key-returning path.
- **Orchard tokens are hashed on the controller → unrecoverable**; rotate = delete+recreate.
  `name` = identity (profile JSON), `token` = secret (Keychain). bootstrap-admin stays on the trunk.
- **App is a separate Xcode target** — `xcodebuild`/`./scripts/dev-app.sh` after GraftCore/CLI
  changes; `swift test` won't catch app breakage ([[feedback_app_target_not_in_swift_build]]).
- **Don't drive Brian's screen with computer-use** — he tests the GUI himself
  ([[feedback_dont_drive_brians_screen]]).
- **SwiftUI traps** (placeholder-as-label, Int→commas, nav-resets-@State, caret-only `Menu("")`,
  `tart exec -i -t`) — [[feedback_graft_swiftui_gotchas]].
